Detailed Description

Interface for materials in the Raytracer.

Defines the contract for how different materials (e.g., diffuse, reflective, refractive, transparent) compute their appearance based on lighting, view direction, and scene context.

Type alias for a function that computes the color of a ray.

This function is used by materials that need to cast secondary rays (e.g., for reflection, refraction) to determine the color contribution from those rays. It takes a ray, the scene's shapes and lights, the camera, and the current recursion depth as parameters, returning the calculated color. The use of std::function allows passing a callable (like a lambda or member function) that encapsulates the core ray coloring logic, often from the Renderer.

Computes the visual appearance (color) of the material at a given surface point.

This is the core method for any material. It determines the material's color based on various factors like surface normal, viewing direction, the point itself, its base color, and interactions with scene lights and other objects (via rayColorFunc for effects like reflection/refraction).

Parameters
normalThe surface normal vector at the hit point.
viewDirThe direction vector from the hit point towards the camera (viewer).
hitPointThe 3D coordinates of the point on the surface being shaded.
colorThe intrinsic base color of the material/surface.
shapesA composite object containing all shapes in the scene, for tracing secondary rays.
lightsA composite object containing all light sources in the scene, for lighting calculations.
cameraThe scene's camera, potentially useful for view-dependent effects.
depthThe current recursion depth for ray tracing (e.g., for reflections/refractions).
rayColorFuncA callback function (typically from the Renderer) to compute the color of secondary rays.
Returns
Math::Vector3D The computed color of the material at the hit point.

Implemented in Raytracer::AMaterials, Raytracer::Reflections, Raytracer::Refractions, and Raytracer::Transparency.
